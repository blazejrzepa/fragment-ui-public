# Changelog

All notable changes to Fragment UI will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.1] - 2025-12-15

### Changed
- Updated package versions to 1.0.1 for npm publishing
- Improved README documentation for all packages
- Updated MCP server README formatting

### Fixed
- Fixed npm publishing workflow
- Resolved version conflicts with existing npm packages

## [1.0.0] - 2025-12-15

### Added
- **@fragment_ui/ui** - React component library with 63+ production-ready components
  - Built on Radix UI primitives
  - Fully accessible (WCAG 2.1 compliant)
  - TypeScript support
  - Dark mode support
  - Tailwind CSS integration

- **@fragment_ui/tokens** - Design tokens package
  - Semantic color system with light/dark themes
  - Consistent spacing scale
  - Typography tokens
  - Elevation and shadow tokens
  - Motion and animation tokens
  - Density modes

- **@fragment_ui/blocks** - Pre-built screen compositions
  - Dashboard layouts
  - Form containers
  - Navigation headers
  - Authentication blocks
  - Pricing tables
  - KPI dashboards
  - And more...

- **@fragment_ui/mcp-server** - MCP server for AI integration
  - Component information and suggestions
  - Code validation
  - Design token access
  - Component code generation
  - Enforcement rules

- **Documentation Site** - Complete documentation portal
  - Component documentation
  - Design tokens reference
  - Theming guide
  - Getting started guides
  - Examples and templates

### Features
- Code-first distribution via registry
- AI-native workflow integration
- Comprehensive documentation
- Type-safe components
- Fully customizable theming
- Dark mode support

## [0.1.0] - 2025-12-15

### Added
- Initial release of Fragment UI public packages
- MCP server for AI integration
- Basic documentation structure
